01 — HTML

HTML Basics & Structure

HTML (HyperText Markup Language) is the skeleton of every web page. It uses tags to wrap content, attributes to add extra info, and nesting to create hierarchy.

📄

DOCTYPE

Tells the browser this is an HTML5 document. Always the first line.

🏗️

Elements

Opening <tag> + content + closing </tag>. Some are self-closing: <img />.

🏷️

Attributes

Key=value pairs inside the opening tag: href, src, class, id, alt.

🧩

Nesting

Elements live inside each other. A child must close before its parent.

index.html — Full skeleton
<!DOCTYPE html>
<html lang="en">
  <head>                             <!-- invisible metadata -->
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1"/>
    <title>Page Title</title>
    <link rel="stylesheet" href="style.css"/>
  </head>
  <body>                             <!-- visible content -->
    <header><h1>Hello World</h1></header>
    <main>
      <p>Content goes here.</p>
    </main>
    <footer><p>Footer</p></footer>
  </body>
</html>
💡
Head vs Body The <head> is invisible — metadata, title, CSS links. The <body> is everything the user sees. The viewport meta tag is essential for responsive design on mobile.
<!DOCTYPE><html><head><body> <title><meta><link><script> <h1>–<h6><p><a><br><hr><span><div>

02 — HTML

Semantic Elements

Semantic tags describe the meaning of the content, not just the appearance. They help browsers, screen readers, and search engines understand your page structure.

semantic-layout.html
<header>    <!-- site header, logo, nav -->
  <nav>       <!-- navigation links -->
    <ul>
      <li><a href="/">Home</a></li>
    </ul>
  </nav>
</header>

<main>         <!-- unique page content -->
  <article>   <!-- standalone, shareable content -->
    <h2>Post Title</h2>
    <section> <!-- thematic grouping inside article -->
      <p>Content…</p>
    </section>
  </article>

  <aside>     <!-- sidebar, tangentially related -->
    <p>Related links</p>
  </aside>
</main>

<footer>       <!-- copyright, links, contact -->
  <p>&copy; 2025</p>
</footer>
TagPurpose
<header>Introductory content — logo, site title, navigation
<nav>Navigation links — main menu, breadcrumbs
<main>The dominant unique content of the page (once per page)
<article>Self-contained, syndicate-able content — blog post, news item
<section>Thematic grouping of content, usually with a heading
<aside>Tangentially related content — sidebar, pull quote
<footer>Footer for a page or section — copyright, links
<figure> / <figcaption>Image with caption
<time>A date/time value machine-readable via datetime attr
<mark>Highlighted / relevant text: like this
<details> / <summary>Expandable disclosure widget
details / summary — built-in HTML accordion
What is semantic HTML?

Semantic HTML means using elements that have inherent meaning — like <article>, <nav>, and <footer> — rather than generic <div> containers for everything. It improves accessibility, SEO, and code readability.

03 — HTML

Forms & Inputs

Forms collect data from users. Each input element has a type that controls the UI and validation. The label element improves accessibility by linking text to its input.

form.html
<form action="/submit" method="post">

  <!-- Text input -->
  <label for="name">Name</label>
  <input type="text" id="name" name="name" placeholder="John Doe" required>

  <!-- Email -->
  <input type="email" placeholder="you@example.com">

  <!-- Password -->
  <input type="password" minlength="8">

  <!-- Number -->
  <input type="number" min="0" max="100" step="5">

  <!-- Dropdown -->
  <select name="role">
    <option value="">Choose role…</option>
    <option value="dev">Developer</option>
  </select>

  <!-- Checkbox -->
  <input type="checkbox" id="agree">
  <label for="agree">I agree</label>

  <!-- Textarea -->
  <textarea rows="4" placeholder="Your message…"></textarea>

  <button type="submit">Submit</button>
</form>

06 — HTML

Media & Global Attributes

HTML can embed images, video, and audio. Global attributes like id, class, data-*, and tabindex work on any element.

media.html
<!-- Image -->
<img src="photo.jpg" alt="Description for screen readers"
     width="800" height="600" loading="lazy" />

<!-- Responsive image with picture -->
<picture>
  <source media="(min-width:800px)" srcset="large.jpg"/>
  <img src="small.jpg" alt="…"/>
</picture>

<!-- Video -->
<video controls width="640" poster="thumb.jpg">
  <source src="video.mp4" type="video/mp4"/>
</video>

<!-- Audio -->
<audio controls>
  <source src="audio.mp3" type="audio/mpeg"/>
</audio>

<!-- Global attributes (work on ANY element) -->
<div id="unique-id"           <!-- unique per page -->
     class="card featured"     <!-- multiple classes -->
     data-user-id="42"        <!-- custom data attribute -->
     title="Tooltip text"     <!-- hover tooltip -->
     hidden                     <!-- hides element -->
     tabindex="0">             <!-- keyboard focusable -->
</div>
💡
alt text is mandatory Every <img> needs an alt attribute. Screen readers read it aloud; search engines index it. If the image is purely decorative, use alt="" (empty string, not missing).

09 — CSS

Specificity & The Cascade

When two rules target the same element, specificity determines which one wins. Think of it as a score — the rule with the higher score is applied.

specificity.css
/* Specificity score: (inline, IDs, classes, elements) */

p               { color: black; }   /* 0,0,0,1 — element */
.text           { color: blue; }    /* 0,0,1,0 — class   */
#intro          { color: red; }     /* 0,1,0,0 — ID      */
/* inline style="color:green"        — 1,0,0,0  highest   */

/* !important overrides ALL — use sparingly */
p               { color: pink !important; }
Element — p
0,0,0,1
Class — .card
0,0,1,0
Class + Element — .card p
0,0,1,1
ID — #hero
0,1,0,0
Inline style
1,0,0,0
⚡
Cascade tiebreaker When specificity is equal, the rule that appears later in the CSS wins. This is the "cascading" in CSS. Always prefer classes over IDs for reusable styles.

10 — CSS

CSS Units

CSS has two types of units — absolute (fixed size) and relative (size relative to something else). Relative units are the key to responsive design.

px

Absolute pixels. Fixed regardless of parent. Good for borders and shadows.

%

Percentage of the parent element's width. Good for fluid widths.

em

Relative to the element's own font-size. 2em = 2× current font.

rem

Relative to the root (<html>) font-size. More predictable than em.

vw / vh

Viewport Width / Height. 100vw = full browser width.

fr

Fractional unit for CSS Grid. 1fr takes 1 "share" of available space.

clamp()

clamp(min, ideal, max) — fluid value that never goes below min or above max.

ch

Width of the "0" character in the current font. Useful for controlling line length.

units.css
:root     { font-size: 16px; }       /* 1rem = 16px */

.container { width: 90%; max-width: 1200px; }

h1  { font-size: clamp(1.8rem, 4vw, 3.5rem); } /* fluid */
p   { font-size: 1rem; max-width: 60ch; }

.hero     { height: 100vh; }         /* full screen */
.card     { padding: 1.5rem; }       /* rem for spacing */
.border   { border: 1px solid #ccc; }/* px for thin borders */

11 — CSS

CSS Variables (Custom Properties)

CSS custom properties let you store values in named variables and reuse them across your stylesheet. Change a variable once → updates everywhere.

variables.css
/* Define on :root = available everywhere */
:root {
  --color-primary: #c84b2f;
  --color-secondary: #2563eb;
  --font-body: 'DM Sans', sans-serif;
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;
  --radius: 8px;
}

.button {
  background: var(--color-primary);
  padding:    var(--spacing-sm) var(--spacing-md);
  border-radius: var(--radius);
}

/* Override locally inside a component */
.dark-theme {
  --color-primary: #ff6b6b; /* only changes inside .dark-theme */
}

/* Fallback value if variable not defined */
p { color: var(--text-color, #333); }
Variables in action

This box uses --brand, --space, and --radius CSS variables. Change the variable → the whole component updates instantly.

💡
JavaScript can read and write CSS variables document.documentElement.style.setProperty('--color-primary', '#ff0000') — this makes dynamic theming (dark mode toggle, etc.) very easy.

12 — Layout

The Box Model

Every HTML element is a rectangular box made of four layers: content → padding → border → margin.

Visual layers
margin (space outside)
border
padding (space inside)
content
box-model.css
/* Apply everywhere — always do this */
*, *::before, *::after { box-sizing: border-box; }

.card {
  width:        320px;
  padding:      20px;           /* all four sides */
  padding:      10px 20px;     /* top/bottom  left/right */
  padding:      5px 10px 15px 20px; /* top right bottom left */
  border:       2px solid #ccc;
  border-radius: 8px;
  margin:       16px auto;      /* center horizontally */
}
💡
border-box vs content-box Default (content-box): width:200px + padding:20px = 240px total. With border-box: the element stays 200px total — padding subtracts from the content. Always use border-box.

14 — Layout

Flexbox

Flexbox is a one-dimensional layout system — it arranges items in a row or column. The parent becomes the flex container; its direct children become flex items.

flexbox.css — Container & Item properties
/* ── Container ──────────────────────── */
.container {
  display:          flex;
  flex-direction:   row;           /* row | column | row-reverse */
  justify-content:  center;       /* main axis: flex-start | center | space-between | space-around */
  align-items:      center;       /* cross axis: stretch | flex-start | center | flex-end */
  flex-wrap:        wrap;         /* wrap onto new lines when needed */
  gap:              16px;         /* space between items */
}

/* ── Items ───────────────────────────── */
.item {
  flex-grow:   1;    /* item stretches to fill space */
  flex-shrink: 0;    /* item won't shrink below flex-basis */
  flex-basis:  200px;/* starting size */
  flex:        1 0 200px; /* shorthand: grow shrink basis */
  align-self:  flex-end; /* override align-items for this item */
  order:       2;    /* visual order (default 0) */
}

15 — Layout

CSS Grid

CSS Grid is a two-dimensional layout system — rows AND columns at the same time. Perfect for page-level layouts and complex card grids.

grid.css
.grid {
  display:               grid;
  grid-template-columns: repeat(3, 1fr);   /* 3 equal columns */
  grid-template-rows:    auto;
  gap:                   16px;             /* row and column gap */
  column-gap:            20px;
  row-gap:               12px;
}

/* Named template areas */
.page-layout {
  display:                    grid;
  grid-template-columns:      200px 1fr;
  grid-template-areas:
    "header  header"
    "sidebar main  "
    "footer  footer";
}
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }

/* Item placement */
.featured {
  grid-column: span 2;   /* 2 columns wide */
  grid-row:    span 2;   /* 2 rows tall */
}

/* Responsive without media queries */
.auto-grid {
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
}

Positioning

The position property controls whether an element follows the normal document flow or is placed manually using top, right, bottom, left.

ValueBehaviour
staticDefault. Follows document flow. top/left do nothing.
relativeOffset from its own normal position. Other elements don't shift.
absoluteRemoved from flow. Positioned relative to nearest non-static ancestor.
fixedRemoved from flow. Stays fixed to the viewport (this nav bar!).
stickyNormal flow, then sticks at a scroll threshold (top:0).

18 — Layout

Overflow

overflow controls what happens when content is larger than its container. It can be set for both axes independently with overflow-x and overflow-y.

overflow comparison

overflow: visible (default)

This text overflows the box boundary and keeps going and going…

overflow: hidden

This text overflows the box boundary and keeps going and going…

overflow-y: scroll

This text overflows the box boundary and keeps going and going — but a scroll bar appears so you can still read it all.

overflow: auto

This text overflows the box boundary and keeps going and going — scrollbar appears only when needed.
overflow.css
overflow:   visible; /* content spills out (default) */
overflow:   hidden;  /* clips content silently */
overflow:   scroll;  /* always shows scrollbars */
overflow:   auto;    /* scrollbars only when needed */
overflow-x: hidden; /* horizontal only */
overflow-y: scroll; /* vertical only */

/* Truncate single-line text with ellipsis */
.ellipsis {
  white-space:   nowrap;
  overflow:      hidden;
  text-overflow: ellipsis;
}

Backgrounds

The background shorthand combines color, image, position, size, and repeat in one line.

backgrounds.css
.box {
  background-color:      #f5f2ed;
  background-image:      url('texture.jpg');
  background-size:       cover;      /* cover | contain | 100% */
  background-position:   center;
  background-repeat:     no-repeat;  /* repeat | repeat-x | repeat-y */
  background-attachment: fixed;      /* parallax effect */
  background-clip:       text;       /* gradient text trick */
}

/* Shorthand */
background: url('bg.jpg') center/cover no-repeat #f5f2ed;

/* Multiple backgrounds stacked */
background:
  linear-gradient(rgba(0,0,0,.4), rgba(0,0,0,.4)),
  url('photo.jpg') center/cover;

filter: brightness(0.9) contrast(1.1) saturate(1.2);

/* Hover effect */
img       { filter: grayscale(1); transition: filter .3s; }
img:hover { filter: none; }

CSS Transforms

transform moves, rotates, scales, or skews an element visually without affecting the document layout. Multiple transforms can be chained.

Transform functions — hover to see outline
↻rotate(15deg)
⊕scale(1.2)
⊘skewX(10deg)
→translate()
↔scaleX(0.7)
✦rotate+scale
transforms.css
transform: rotate(15deg);
transform: scale(1.5);             /* scale(x, y) */
transform: translate(20px, -10px); /* move x, y */
transform: translateX(50%);        /* 50% of own width */
transform: skewX(15deg);
transform: skewY(5deg);

/* Chain multiple: applied right to left */
transform: rotate(-5deg) scale(1.1) translateY(-4px);

/* 3D transforms */
transform: rotateY(180deg);    /* flip card effect */
transform: perspective(500px) rotateX(10deg);

/* transform-origin: pivot point (default: center) */
transform-origin: top left;
transform-origin: 50% 100%; /* bottom center */

/* Common centering trick */
.center {
  position:  absolute;
  top:       50%;
  left:      50%;
  transform: translate(-50%, -50%);
}

Transitions

Transitions animate a CSS property from one value to another when it changes (usually on :hover or via JavaScript). They require a start state and an end state.

Hover this box
Hover me — watch colour, shape & scale change
transitions.css
/* transition: property duration timing-function delay */

.btn {
  background:  var(--accent);
  transition:  background .3s ease;          /* one property */
  transition:  all .3s ease;                 /* all properties */
  transition:  background .3s, transform .2s; /* multiple */
}
.btn:hover { background: var(--accent2); transform: scale(1.05); }

/* Timing functions */
transition-timing-function: ease;         /* slow-fast-slow (default) */
transition-timing-function: linear;       /* constant speed */
transition-timing-function: ease-in;      /* slow start */
transition-timing-function: ease-out;     /* slow end */
transition-timing-function: ease-in-out;
transition-timing-function: cubic-bezier(0.68,-0.55,0.27,1.55); /* bounce */
💡
Performance tip Only animate transform and opacity for smooth 60fps animations. Animating width, height, margin, or top/left triggers layout recalculation and can cause jank.

CSS Animations

Animations use @keyframes to define states over time, then the animation property to apply them. Unlike transitions, animations can loop, reverse, and run automatically.

Live animations (running continuously)
💗pulse
⚙️spin
🏀bounce
→slide-in
⚠️shake
animations.css
/* 1. Define keyframes */
@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50%       { transform: translateY(-20px); }
}

@keyframes fadeIn {
  from { opacity: 0; transform: translateY(15px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes spin {
  to { transform: rotate(360deg); }
}

/* 2. Apply the animation */
.loading {
  animation:            spin;       /* name */
  animation-duration:   1s;         /* how long */
  animation-timing-function: linear;
  animation-iteration-count: infinite; /* loop forever */
  animation-direction:  normal;     /* normal | reverse | alternate */
  animation-fill-mode:  both;       /* keeps final state */
  animation-delay:      0.2s;       /* wait before starting */
}

/* Shorthand */
animation: fadeIn .5s ease .2s both;  /* name dur timing delay fill */

/* Staggered entrance (animation-delay per child) */
.item:nth-child(1) { animation-delay: 0s; }
.item:nth-child(2) { animation-delay: .1s; }
.item:nth-child(3) { animation-delay: .2s; }

responsive.css
/* ── Mobile First approach ─────────────────── */
/* Write base styles for mobile, then expand */

.grid {
  display: grid;
  grid-template-columns: 1fr;    /* 1 column on mobile */
  gap: 1rem;
}

@media (min-width: 640px) {           /* tablet and up */
  .grid { grid-template-columns: 1fr 1fr; }
}

@media (min-width: 1024px) {          /* desktop */
  .grid { grid-template-columns: repeat(3, 1fr); }
}

/* ── Dark Mode ───────────────────────────── */
@media (prefers-color-scheme: dark) {
  body  { background: #1a1917; color: #e8e2d9; }
  .card { background: #252320; border-color: #333; }
}

/* ── Accessibility ───────────────────────── */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}

/* ── Print ───────────────────────────────── */
@media print {
  .nav, .footer { display: none; }
  body { font-size: 12pt; color: black; }
}

/* ── Flexible without media queries ─────── */
.auto-grid {
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
}
h1 { font-size: clamp(1.8rem, 4vw, 3.5rem); }
p  { width: min(640px, 90%); }
