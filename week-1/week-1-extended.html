01

HTML foundation

A webpage begins with a correct document skeleton

The first step is learning that a page is not just visible text. It has a document type, language, metadata, viewport settings, a title, and then structured body content. 

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>My first structured page</title>
  </head>
  <body>
    <header>Page introduction</header>
    <main>Main content</main>
    <footer>Footer content</footer>
  </body>
</html>
02

Meaningful content

Semantic HTML makes content understandable

Before design, HTML should communicate meaning. Headings, paragraphs, lists, links, and definition lists create a page that is readable and accessible.

Mini article example

Semantic HTML helps browsers, search engines, screen readers, and developers understand what each part of the page is for.

  • Headings create a page outline.
  • Paragraphs group related ideas.
  • Lists make steps easier to scan.
  • Links connect pages and sections.

Core semantic elements

<header>
Introductory content for a page or section.
<main>
The main unique content of the current page.
<section>
A themed group of related content.
<article>
A self-contained piece of content.

Forms and accessibility

Forms need labels, types, focus states, and grouping

Good forms are not just inputs. They use labels, correct input types, fieldsets, required states, placeholders, and visible keyboard focus.
